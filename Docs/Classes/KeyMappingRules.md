# KeyMappingRules

## Назначение
Класс `KeyMappingRules` предназначен для формирования таблицы (DataTable) с правилами маппинга внешних ключей (связей) между загруженными таблицами (листами Excel-файла). Он определяет, на какие таблицы и столбцы ссылаются определенные поля в главной таблице (`РядыРигелейXXX`).

## Зависимости
- `System.Data.DataTable`
- `System.Text.RegularExpressions.Regex`
- `System.Windows.Forms.MessageBox`
- Получает на вход список имен загруженных таблиц (`IEnumerable<string>`).

## Основные методы
- `GetMappingRules(IEnumerable<string> loadedTableNames)`: 
  Проходит по списку загруженных таблиц, находит таблицы, соответствующие паттерну `РядыСтоек(.*)`, и формирует для них правила маппинга. Возвращает `DataTable` с колонками:
  - `MainTable` - имя исходной таблицы (например, `РядыРигелейXXX`).
  - `ColumnName` - имя столбца в исходной таблице, который является внешним ключом.
  - `DataType` - тип данных ключа (`int/float`).
  - `Description` - описание поля.
  - `TargetTable` - целевая таблица для связи (например, `РядыСтоекXXX`, `РядыЗаполненийXXX`).
  - `TargetColumn` - целевой столбец в целевой таблице. Для `ТипРядовСтоек` это конкретный столбец `тип`. Для остальных (`ТипРядовЗаполнений`, `ТипКреплений`, `Подпорки`, `Опоры`) используется значение `{Value}`, которое означает, что значение ячейки в главной таблице является именем целевого столбца в целевой таблице. При этом индекс строки в целевой таблице определяется как `ТипРядовСтоек - 1`.

## Правила маппинга
В соответствии со структурой базы данных настраиваются следующие связи от главной таблицы `РядыРигелейXXX`:
- `ТипРядовСтоек` -> `РядыСтоекXXX` (столбец: `тип`)
- `ТипРядовЗаполнений` -> `РядыЗаполненийXXX` (столбец: по значению ячейки)
- `ТипКреплений` -> `РядыКрепленийXXX` (столбец: по значению ячейки)
- `Подпорки` -> `РядыКрепленийXXX` (столбец: по значению ячейки)
- `Опоры` -> `РядыОпорXXX` (столбец: по значению ячейки)

**Важно:** Метод также проверяет существование целевых таблиц (`TargetTable`) в переданном списке `loadedTableNames`. Если хотя бы одна из требуемых таблиц отсутствует, пользователю выводится предупреждение через `MessageBox`.
