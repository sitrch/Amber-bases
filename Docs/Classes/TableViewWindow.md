# TableViewWindow

**Модуль:** UI  
**Расположение:** `AmberBases/UI/TableViewWindow.xaml` и `AmberBases/UI/TableViewWindow.xaml.cs`

## Назначение
Модальное окно-обёртка для `DictionaryTableControl`. Открывается при клике "Открыть таблицу [Родительская]" в контекстном меню и поддерживает вложенную навигацию — при клике на FK ячейку внутри таблицы открывается новая `TableViewWindow`. Поддерживает сохранение, отмену и закрытие с проверкой несохранённых изменений.

## Логика работы
- Принимает тип сущности, коллекцию данных и словарь всех коллекций.
- Создаёт локальные копии коллекций (`ObservableCollection`) для изоляции данных.
- Размещает `DictionaryTableControl` с callback для открытия родительской таблицы в новом окне.
- Каждое окно вложено в предыдущее (`Owner`), что обеспечивает стек окон при навигации.
- При закрытии окна данные в родительском окне обновляются через `LoadData()`.
- Отслеживает изменения через снимки коллекций и предоставляет три кнопки действий внизу окна.

## Структура UI
```
+-------------------------------------------+
| [Заголовок: "Таблица: ..."]              |
+-------------------------------------------+
|                                           |
|           DictionaryTableControl          |
|                                           |
+-------------------------------------------+
|                [Сохранить] [Отменить] [Закрыть]  |
+-------------------------------------------+
```

## Кнопки действий
- **Сохранить** — сохраняет все изменения во все коллекции справочников в БД через `IDictionaryDataService`. Обновляет снимок оригинальных данных.
- **Отменить** — сбрасывает все изменения и восстанавливает данные из БД (переинициализирует `DictionaryTableControl`).
- **Закрыть** — закрывает окно. Если есть несохранённые изменения, показывает диалог с тремя вариантами:
  - **Да** — Сохранить и закрыть
  - **Нет** — Отменить изменения и закрыть
  - **Отмена** — Остаться в редакторе

## Ключевые методы

### Конструктор и инициализация
- `TableViewWindow(Type entityType, IList collection, Dictionary<Type, IEnumerable> allCollections)`: Конструктор. Инициализирует сервис данных, локальные коллекции и снимок оригинальных данных.

### Отслеживание изменений
- `CloneAllCollections()`: Создаёт снимок всех коллекций для сравнения.
- `CloneCollection(IList collection)`: Клонирование коллекции объектов в список словарей.
- `CheckForChanges()`: Проверяет, есть ли изменения в коллекциях по сравнению с оригиналом.
- `HasCollectionChanged(string key, IList currentCollection)`: Проверяет изменения в конкретной коллекции.

### Сохранение и отмена
- `SaveChanges()`: Сохраняет изменения в БД через Sync-методы.
- `DiscardChanges()`: Отменяет изменения, перезагружая данные из БД.
- `RestoreOriginalValues()`: Восстанавливает оригинальные значения коллекций из БД.

### Sync-методы (сохранение в БД)
- `SyncSystemProviders()`, `SyncProfileSystems()`, `SyncColors()`, `SyncWhipLengths()`, `SyncProfileTypes()`, `SyncApplicabilities()`, `SyncProfileArticles()`, `SyncCoatingTypes()`: Синхронизируют локальные коллекции с БД (добавление новых, обновление существующих, удаление удалённых записей).

### Навигация
- `OpenParentTable(object parentItem, Type parentType)`: Открывает новое окно `TableViewWindow` для родительской сущности.
- `GetCollectionForEntityType(Type type)`: Возвращает локальную коллекцию для типа сущности.
- `GetDisplayName(string typeName)`: Возвращает русифицированное название сущности.

### Обработчики событий
- `BtnSave_Click`: Обработчик кнопки "Сохранить".
- `BtnCancel_Click`: Обработчик кнопки "Отменить".
- `BtnClose_Click`: Обработчик кнопки "Закрыть".
- `TableViewWindow_Closing`: Обработчик события `Closing` — показывает диалог с несохранёнными изменениями.
