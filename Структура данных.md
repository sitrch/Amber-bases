# Структура данных базы данных фасада "стойка-ригель"

## Обзор
База данных описывает конструкцию фасада многоэтажного дома с 4 плоскостями.
Название файла соответствует названию плоскости (например, `_Шаблоны(5-1)-(5-6).xlsx`).

## Таблицы

### 1. РядыРигелей{XXX} — Основная таблица
**Описание:** Содержит данные о рядах ригелей для каждого этажа.

| Столбец | Тип | Описание | Связь |
|---------|-----|----------|-------|
| Этаж | int | Номер этажа | |
| НизСтойки | float | Высота низа стойки от нулевой отметки (мм) | |
| ВысотаСтойки | float | Высота стойки (мм) | |
| Зазор | float | Тепловой зазор между стойками (мм) | |
| РядИзделия | int | Номер ряда ригеля от низа стойки | |
| РядФасада | int | Номер ряда от нулевой отметки | |
| ПоЦентрам | float | Межцентровое расстояние между ригелями (мм) | |
| Установка | float | Высота установки ригеля от нулевой отметки (мм) | |
| ТипРядовСтоек | int/float | ID типа ряда стоек | FK → РядыСтоек{XXX}.тип |
| ТипРядовЗаполнений | string | Тип заполнения | FK → РядыЗаполнений{XXX}.Зап* |
| ТипКреплений | string | Тип крепления ригеля | FK → РядыКреплений{XXX}.Креп* |
| Подпорки | string | Тип подпорки под ригель | FK → РядыКреплений{XXX}.Подпорка* |
| Опоры | string | Тип опоры под заполнение | FK → РядыОпор{XXX}.Опора* |
| Уровень ЧП | float | Уровень чистого пола этажа (мм) | |
| Низ Окна | float | Высота низа окна от уровня чистого пола (мм) | |
| РучкаотНизаОкна | float | Расстояние ручки от низа окна (мм) | |

**Примечание:** Строки с маркерами (ВС, Зап1, Зап2, Межэтажка, СН) в столбцах `ТипРядовЗаполнений`, `ТипКреплений` и др. являются ключами для lookup-таблиц.

---

### 2. РядыСтоек{XXX}
**Описание:** Параметры расстановки стоек по плоскости.

| Столбец | Тип | Описание |
|---------|-----|----------|
| Плоскость | string | Название плоскости (напр., `(5-1)-(5-6)`) |
| тип | int | ID типа ряда стоек (соответствует `ТипРядовСтоек` в `РядыРигелей`) |
| стойка | int | Порядковый номер стойки |
| поЦентрам | float | Расстояние от предыдущей стойки (мм) |
| Установка | float | Накопленное расстояние от начала (мм) |
| ДопСтойка | float | Флаг дополнительной стойки (1/NaN) |

---

### 3. РядыЗаполнений{XXX}
**Описание:** Lookup-таблица типов заполнений (панелей, остекления).

| Столбец | Тип | Описание |
|---------|-----|----------|
| Стойка | int | ID стойки (индекс строки) |
| Зап1 | string | Тип заполнения для Зап1 |
| Зап1_20 | string | Тип заполнения для Зап1_20 |
| Зап2 | string | Тип заполнения для Зап2 |
| Зап2_57 | string | Тип заполнения для Зап2_57 |
| Межэтажка | string | Тип заполнения для межэтажного участка |
| Базовый | string | Базовый тип заполнения |
| ЗапНизВерх | string | Тип заполнения низ/верх |
| Термомост | string | Наличие термомоста (Да/Нет/NaN) |
| ВС | string | Тип заполнения ВС (все стойки) |

**Ключи:** Значения из `РядыРигелей.ТипРядовЗаполнений` соответствуют названиям столбцов этой таблицы.

---

### 4. РядыКреплений{XXX}
**Описание:** Lookup-таблица типов креплений ригелей и подпорок.

| Столбец | Тип | Описание |
|---------|-----|----------|
| пл1_Зап1 | string | Тип заполнения для плоскости 1, Зап1 |
| пл1_Зап2 | string | Тип заполнения для плоскости 1, Зап2 |
| Стойка | int | ID стойки (индекс строки) |
| Креп1 | string | Тип крепления для Зап1 |
| Креп1_20 | string | Тип крепления для Зап1_20 |
| Подпорка1 | string | Тип подпорки для Зап1 |
| Подпорка1_20 | string | Тип подпорки для Зап1_20 |
| Креп2 | string | Тип крепления для Зап2 |
| Креп2_57 | string | Тип крепления для Зап2_57 |
| Подпорка2 | string | Тип подпорки для Зап2 |
| Подпорка2_57 | string | Тип подпорки для Зап2_57 |
| КрепМ | string | Тип крепления для Межэтажка |
| КрепМ_Ус | string | Тип крепления для Межэтажка усиленный |
| ВС | string | Тип крепления для всех стоек |

**Связь:** 
- `РядыРигелей.ТипКреплений` → столбцы `Креп*` этой таблицы
- `РядыРигелей.Подпорки` → столбцы `Подпорка*` этой таблицы

---

### 5. РядыОпор{XXX}
**Описание:** Lookup-таблица типов опор под заполнения.

| Столбец | Тип | Описание |
|---------|-----|----------|

### 6. ГраницыИзделий{XXX}
**Описание:** Маппинг стоек на изделия (панели).

| Столбец | Тип | Описание |
|---------|-----|----------|
| Стойка | int | Номер стойки |
| ЛеваяСтойка | int | Номер левой границы изделия |
| ПраваяСтойка | int | Номер правой границы изделия |
| НомерИзделия | int | ID изделия |

---

### 7. Изделия{XXX}
**Описание:** Список изделий (панелей) по этажам и стойкам.

| Столбец | Тип | Описание |
|---------|-----|----------|
| Этаж | int | Номер этажа |
| с1, с5, с9, ... | string | Марка изделия для соответствующей стойки |
| Изделия на этаже | string | Список всех изделий на этаже (через запятую) |

---

### 8. Размещение изделий на этажах{XXX}
**Описание:** Группировка этажей с одинаковым набором изделий.

| Столбец | Тип | Описание |
|---------|-----|----------|
| Этаж | int | Базовый номер этажа |
| Этажи с сопадающим набором устанавливаемых изделий | string | Описание групп этажей (напр., "нечётные 3,5,...,55") |
| Изделия | string | Список изделий для группы этажей |
| Кол-во этажей | int | Количество этажей в группе |

---

### 9. Подсчёт изделий{XXX}
**Описание:** Сводная таблица количества изделий по рядам и этажам.

| Столбец | Тип | Описание |
|---------|-----|----------|
| РядИзделий | int | Номер ряда изделий |
| Этаж | int | Номер этажа |
| Изделие | string | Марка изделия |
| Количество | int | Количество изделий |

---

### 10. Data
**Описание:** Справочная таблица маппинга типов элементов на конкретные значения.

| Столбец | Тип | Описание |
|---------|-----|----------|
| Заполнения | string | Код заполнения (1а, 1б, 3а, ...) |
| ТипРядовЗаполнений | string | Имя типа (Зап1, Зап2, Межэтажка, ...) |
| ТипКреплений | string | Имя типа крепления (Креп1, Креп2, КрепМ, ...) |
| Подпорки | string | Имя типа подпорки (Подпорка1, Подпорка2, ...) |
| Опоры | string | Имя типа опоры (о1, о2, о3, ...) |
| Термомосты | string | Код термомоста (т0, т1, т2, ...) |
| Крепление ригеля | string | Код крепления (с1, с2, с3, ...) |
| Подставка | string | Тип подставки |
| Опоры.1 | string | Дополнительный тип опоры |
| Ряды опор | string | Имя ряда опор (ОпораМ, Опора1, ...) |

## Зависимости между таблицами

```
РядыРигелей{XXX}
    ├── ТипРядовСтоек → РядыСтоек{XXX}.тип
    ├── ТипРядовЗаполнений → РядыЗаполнений{XXX}.{column}
    ├── ТипКреплений → РядыКреплений{XXX}.{column}
    ├── Подпорки → РядыКреплений{XXX}.{column}
    └── Опоры → РядыОпор{XXX}.{column}

ГраницыИзделий{XXX}
    └── НомерИзделия → Изделия{XXX}.с{N}

Размещение изделий на этажах{XXX}
    └── Группирует этажи с одинаковыми изделиями

Подсчёт изделий{XXX}
    └── Агрегирует данные из Изделия{XXX}

Data
    └── Справочник для расшифровки кодов типов
```

## Вопросы для уточнения

1. **Соответствие ключей lookup:** Как точно сопоставляются значения из `РядыРигелей.ТипРядовЗаполнений` (например, "Зап1", "Межэтажка") со столбцами в `РядыЗаполнений`? По точному совпадению имени столбца?

2. **Индексация строк в lookup-таблицах:** Столбец `Стойка` в таблицах `РядыЗаполнений`, `РядыКреплений`, `РядыОпор` — это индекс строки (0-based) или связано с `ТипРядовСтоек` из `РядыРигелей`?

3. **Плоскости:** Файлы с префиксом `_Шаблоны` содержат данные для одной плоскости. Как связаны данные между разными плоскостями одного дома?

4. **Data таблица:** Каково точное назначение таблицы `Data`? Это справочник для расшифровки кодов или для генерации спецификаций?

5. **РядыРигелей структура:** В таблице `РядыРигелей` есть строки-заголовки повторения (содержат "Этаж", "Ригель", "чп" и т.д.). Это артефакты экспорта из Excel или имеют смысловую нагрузку?
